# Repurposing Chain

> One idea enters. N platform-native pieces come out. Each one THINKS about the topic differently — this is NOT reformatting. If someone followed Juan on all three platforms, they should not be annoyed seeing the same idea reused.

## Why Juan's Chain is Long-Form-First (Not X-First)

The article recommends writing X first because brevity forces clarity. For Juan, that's wrong. His signature move is the **Economics Reveal** — math that needs room to breathe before it can be compressed. Starting on Twitter strips the proof; starting on blog or LinkedIn builds it, then Twitter becomes the compression.

**Exception:** Hot takes, Contrarian Premises, and standalone observations can originate on Twitter and expand outward.

## The Two Chain Paths

### Path A: Framework-Heavy Topic (default)

Use when the topic involves a framework, formula, or Economics Reveal.

**Step 1: [[blog]] (Seed)**
Write the full framework with math, subheadings, inline formulas, comparison tables. 1,500-3,000 words.

**Step 2: [[linkedin]] (Narrative)**
Extract one framework or one Economics Reveal. Add personal narrative framing ("3 months ago I was diagnosing a brand that..."). 1,300-2,000 chars. Different angle: the story, not the system.

**Step 3: [[twitter]] (Compression)**
Distill the core framework into a 5-8 tweet thread. Or extract one contrarian claim into a single tweet. Different angle: the punchline, not the proof.

### Path B: Hot-Take Topic

Use when the topic is a single contrarian insight or observation.

**Step 1: [[twitter]] (Seed)**
Write the contrarian single or short thread. Force brevity.

**Step 2: [[linkedin]] (Expand)**
Add narrative context, personal framing, and the Economics Reveal behind the take. Different angle: why it matters, not just what's wrong.

**Step 3: [[blog]] (Deepen — optional)**
Only if the take earns an audience. Turn into a full framework piece. Different angle: the full system replacing what's broken.

## What Actually Changes Between Platforms

| Element | How It Adapts |
|---------|---------------|
| Angle | Different entry point into the same topic (system vs story vs punchline) |
| Hook | Rewritten per [[hooks]] and [[platform-tone]] |
| Tone | Rigorous (blog) → narrative (LinkedIn) → compressed (Twitter) |
| Format | Long-form → standard post → thread/single |
| Length | 2,000 words → 1,500 chars → 280 chars |
| Depth | Full proof → one proof point → punchline |

## The Litmus Test

Before publishing, ask:
- **If someone followed me on all three platforms, would they be annoyed seeing the same thing everywhere?**
- If YES → you're reformatting. Go back and find a different angle.
- If NO → you've made 3 unique pieces from one idea. Ship.

## Batch Workflow

1. Pick 1 framework-heavy topic + 1 hot-take topic per week
2. Run Path A for the first, Path B for the second
3. Write all platform versions in one session — don't spread across days
4. Review all outputs together before publishing any
5. Schedule using cadence from [[content-strategy]]
